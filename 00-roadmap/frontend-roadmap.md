# Frontend Roadmap

This roadmap is designed for internship preparation and focuses on what I need to learn, not how fast I finish it.

## Main Goal

By the end of this roadmap, I should be able to:

- build solid interfaces with HTML, CSS, JavaScript, TypeScript, React, and TailwindCSS
- build polished React-based frontend projects
- explain core frontend concepts clearly in interviews
- debug common UI and state issues with confidence
- write cleaner, more maintainable frontend code
- show practical evidence of learning through notes and projects

## Learning Path

### 1. HTML, CSS, and JavaScript Foundations

Focus:

- refresh core frontend building blocks
- strengthen layout and styling confidence
- review JavaScript that shows up constantly in frontend work

Topics:

- semantic HTML
- forms and input elements
- accessibility basics in HTML
- CSS box model
- flexbox
- grid
- responsive design basics
- positioning
- CSS specificity
- modern JavaScript syntax
- ES6+ syntax used daily
- array methods and object manipulation
- async JavaScript basics

Deliverables:

- write HTML, CSS, and JavaScript review notes in `01-foundations/`
- build a responsive static page or landing page
- practice recreating one simple layout with clean HTML and CSS

Interview outcomes:

- explain semantic HTML and why it matters
- explain flexbox vs grid at a high level
- explain JavaScript concepts like scope, closures, and async behavior

### 2. TypeScript Fundamentals for Frontend

Focus:

- add type safety on top of JavaScript fundamentals
- understand the TypeScript features most useful in frontend work
- get comfortable reading and writing typed code

Topics:

- basic TypeScript types
- type inference
- interfaces and type aliases
- union types
- function typing
- typing objects and arrays
- optional properties
- narrowing
- typing component props

Deliverables:

- write TypeScript notes in `01-foundations/`
- convert a few JavaScript examples into TypeScript
- practice defining types for simple frontend data shapes

Interview outcomes:

- explain why TypeScript is useful in frontend projects
- explain the difference between interfaces and type aliases at a high level
- explain how TypeScript helps catch bugs earlier

### 3. React Fundamentals and Component Thinking

Focus:

- move from vanilla frontend thinking into React thinking
- understand how components organize UI
- build confidence with state-driven interfaces

Topics:

- React components
- JSX
- props
- state
- event handling
- conditional rendering
- list rendering
- forms
- lifting state up
- component composition

Deliverables:

- write React fundamentals notes in `02-frontend/`
- build a small React app such as a counter, to-do list, or habit tracker
- refactor part of a UI into reusable components

Interview outcomes:

- explain the difference between props and state
- explain how React re-renders
- explain controlled components

### 4. React Data Flow, TailwindCSS, and App Structure

Focus:

- move from toy components to small real apps
- learn a fast, practical styling workflow
- understand how data flows through a frontend app

Topics:

- local state vs shared state
- `useEffect`
- data fetching
- loading, error, and empty states
- form handling
- basic custom hooks
- folder structure for React apps
- TailwindCSS utility classes
- responsive styling with TailwindCSS
- reusable UI patterns with TailwindCSS

Deliverables:

- write notes on `useEffect`, data fetching, and TailwindCSS
- build a small app that fetches data from a public API
- style the app with TailwindCSS and clean loading or error states

Interview outcomes:

- explain when `useEffect` runs
- explain how to fetch data safely in a React app
- explain why TailwindCSS can be useful in frontend projects

### 5. Routing, Testing, Polish, and Interview Prep

Focus:

- turn learning into portfolio-quality evidence
- prepare to talk about frontend work in interviews

Topics:

- React Router basics
- page-level structure
- accessibility review
- semantic HTML review
- keyboard interaction
- performance basics
- avoiding unnecessary re-renders
- frontend testing basics
- component testing mindset
- debugging workflow
- network and UI debugging
- project cleanup and refactoring
- README writing
- internship-style frontend interview review

Deliverables:

- write notes on routing, testing, and debugging basics
- polish one frontend mini project into a showcase project
- add a project write-up in `10-projects/`
- create an interview review summary in `08-interview-prep/`

Interview outcomes:

- explain client-side routing
- explain how you debug a broken UI
- explain how you would test a component
- walk through one frontend project clearly from problem to implementation

## Output Rule

For each learning section, I should aim to produce:

- at least 2 note files
- at least 1 practice build or feature
- at least 1 short reflection on what felt confusing

## Success Checklist

By the end of this roadmap, I should have:

- notes on HTML, CSS, JavaScript, TypeScript, React, and TailwindCSS
- React fundamentals notes
- notes on state, effects, routing, accessibility, and testing
- one polished frontend mini project
- one API-based project or feature
- clearer interview explanations for common frontend topics
